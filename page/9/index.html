SDRAM + CPLD TFT驱动器的PCB布局失误与经验

SDRAM走线原则

  1. SDRAM走线应尽可能短，尽可能直，尽量采用折线风格而不是自然曲线风格。
  2. SDRAM的数据、地址走线长度应尽可能在7CM内，长度差不走过2CM。
  3. SDRAM的时钟线应短于所有的数据线和地址线，因为在SDRAM的时钟相位可以通过FPGA/CPLD内部的锁相环来进行调整，等效于加长SDRAM的时钟线，而地址线和数据线则不能。
  4. SDRAM的走线间距应尽量遵守3W原则。
  5. SDRAM的时钟线应实现包地。
  6. SDRAM时钟线可以串一个几十欧的小电阻。

用户接口总线走线原则

  1. 走线尽可能短直，尽量采用折线风格而不是自然曲线风格。
  2. 走线长度尽可能等长，长度差与总线的频率有关。
  3. 数据线串联几十欧的电阻。
  4. CS、RS、WR、RD线要尽量远离数据线，用地线隔离，且这四根控制线之间也要实现包地。
  5. RST复位线尽可能远离数据线，同时也应远离频率较高的WR、RD线。
  6. RST线上可串一个103电容，以滤除从其它线上感应到的信号，以确保控制器不会无故复位。

CPLD/FPGA布线原则

  1. PLL专用输入引脚应加磁珠和电容滤波。
  2. 每一组电源脚都应就近放置退耦电容。
  3. 芯片底部尽可能保证完整的地线。
  4. 芯片的电源线尽可能粗、短、直，尽量不要为了省事走成环形，环形的VCC线容易感应信号，从而将噪所引入电源，对其它芯片及电路造成干扰。

其它EMC相关请遵循一般原则，比如尽可能保证地电阻较小，保证地平面完整，以及地电阻的均匀。

wordpress 模板修改注意事项

模模中各文件的作用

wordpress中的定义：

文章——就是博客文章，通常用分类。

页面——就是类似于关于之类的页面，通常无分类。

=======================================================================================

content.php 文件是首页的内容生成

content-single.php 文件是生成每篇独立文章的内容

content-page.php 文件是生成页面的内容

single.php 文件是生成每篇独立文章

page.php 文件是生成每个WP页面

content.php 文件是生成评论

archive.php 文件是生成归档

footer.php 文件是生成页脚，比如IPC备案信息

function.php 这个文件功能很强大，比如wiget或者siderbar的参数都可以在这里设置

header.php 文件是页面头部生成，比如网站标题，“关于”菜单等

image.php 图片生成

index.php 默认主题主页

no-results.php 没有找到要搜索的内容

search.php 搜索页面

searchform.php 生成搜索结果列表

sidebar.php 小工具条生成，里面有些信息可以在functions.php中通过fittler的形式修改，优先级更高。

404.php 生成网址找不以的404错误页

inc文件夹 里面放置要包含的一些自定义文件

js文件夹 放置主题JS文件

languages 放置语言本地化模版文件

=======================================================================================

inc文件夹

custuom-header.php——也是修改网页头部的，但比header.php的优先级高。

template-tags.php——中存放了导航条生成，评论生成等相关功能函数。

其它文件目前我也不清楚。

《黑客与画家》读书笔记

本文全凭个人记忆和《黑客与画家》一书目录写成，该书的精彩之处我并未能一一道来，感兴趣的还是自己花上一个星期的业余时间读完为好。

《黑客与画家》这本书的作者本身是一个传奇，而造就这种传奇的原因除了运气，还与他个人的思想息息相关——对任何事情形成自己的思考，不盲从。

全书关于语言的那几张，我觉得写得不是太好，也可能是我不感兴趣，不过让我对LISP更感兴趣了，哪天学一学函数式编程。另外荐言中对本书所作的所谓程序员文化感觉不太符，与其说是程序员文化不如说是作者的个人思维，毕竟这是一本文集，很多篇博文收录在一起原本并无系统性，不太能达主题上的一致性。好吧，那我就只谈谈我感深趣的。

为什么书呆子不受欢迎

这一章所描述的场景在中国的中学里面简直是一样一样的，甚至有过之怕无不及。我之前从来没想过这个问题，从来没有，在中国你周围的人只会对这种事情用一些最俗的话来回答这个问题，比如说你情商不够高，不够合群，不知道怎么跟人打交道。保尔对此问题的解答提供了一个有趣的方向，他用社会属性来看待校园的群体关系。如豆瓣上所述，“他们的心思在别的地方”，这一点就像《爸爸去哪儿》里的小孩绝大多数会喜欢玩游戏，而胡军的小孩却觉得那些游戏太幼稚了。

黑客与画家

这一章也比较有趣，当然他所指的黑客是不是我们常说的脚本坏小子或者是黑客工具仔，他指的是牛逼轰轰的专家级计算机高手。在他的眼里，黑客、画家、建筑师都是进入化境的手艺人，用心雕琢，喜欢创造，喜欢思考，喜欢继承喜欢打破常规。里面谈到了“白天工作”，“通过实践学习创作”等一些现在比较流行的思维。

不能说的话

这一章所述的无非是怎么找出哪些话不能说，为什么不能说，你该怎样对待这些不能说的话，处理方式跟中国的中庸之道一样“闭口不谈”、“祸从口出”。然后进一步就是如王圣人王阳明老先生所传世的“知行合一”。

良好的坏习惯

这一篇里面讲到黑客就是打破常规，对现有的东西不满，然后动手改造的一群人。感觉有点像谈言论自由，思想解放。说实话，有点愤青的感觉。

如何创造财富

如何创造财富？作者给出的答案就是制作对用户有价值的产品，或者说创业吧，虽然二者并不等价。当然里面也说到了创业也需要运气，但值得去尝试。
我最喜欢其中“工作是什么”、“更努力地工作”并不见得能致富等几小节的内容，还有“可测量性和可放大性”里面说到拒决平庸。里面提到

// 取表格值
BOOL CExcel::GetCell(int nRow, int nCol, CStdString &amp;strContent)
{
    try
    {
        // 移动到指定位置
        strContent.Empty();
        m_pRecordset-&gt;MoveFirst();
        m_pRecordset-&gt;Move(nRow);

        // 取值
        _variant_t var;
        CStdString strF;
        strF.Format(_T("F%d"), nCol+1);
        var = m_pRecordset-&gt;GetCollect(_variant_t((long) (nCol)));
        //if( var.vt != VT_NULL)
        //  strContent = (LPCTSTR)_bstr_t(var);

        switch(var.vt)
        {
        case VT_R8:        //8字节的数字
            strContent.Format(_T("%d"),(int)var.dblVal);
            break;
        case VT_BSTR:        //字符串
            strContent = var.bstrVal;
            break;
        case VT_I4:
            strContent.Format(_T("%d"), (int)var.lVal);
            break;
        case VT_EMPTY:        //单元格为空
        case VT_NULL:
            strContent = _T("");
            break;
        default:
            break;
        }

    }
    catch (_com_error e)
    {
        return false;
    }

    return true;
}

BOOL CExcel::ReadRecordCount(int&amp; nCount)
{
    CStdString strText;
    if(!GetCell(1, 1, strText)) return FALSE;
    int n = strText.Find(_T(", "));
    if(n &lt; 0) return FALSE;
    strText = strText.Mid(n+2, strText.GetLength()-n-2);
    nCount = _tstoi(strText);
    return TRUE;
}

    return true;
}

在Altium Designer中快速作制原理图封装

原理图封装制作最麻烦的有两点：

  1. 画引脚
  2. 填写引脚电气定义

我常看见一些PCB工程师画了多年PCB，用了多年的AD（好吧，我承认AD确实在某些方面不如CANDENCE，甚至在PCB LAYOUT工程师中也受歧视）却仍然采用的着最累和最麻烦的办法，难道你们就不曾浏览过AD每次升级有哪些新特性吗，尽管有些根本算不上新特性。我们来看看通常是怎么制作原理图封装的。
如下所示是一个SDRAM的引脚定义，共有54个脚。

首先在SCH LIB图纸里按快捷键PP。

然后按住SHIFT拖出9个（或者其它数字）相同的PIN，它会自增定义。

然后框选这9个PIN，同样按信SHIFT拖出下一组9个PIN，依次类推拖出余下
的PIN。在最开始拖第一组9PIN的过程中我们得保证间距一至。

接着将引脚与封装主体对齐

接着一一修改它的引脚号有电气信号名称。

仅这54个脚就够折磨人了，如果你用DDR，动辄上百个脚的的BGA，用上面这种办法你得累死。
事实上只需以下几步无论多少引脚就搞定：

  1. 按PP画出一个引脚
  2. 按CTRL+C复制画出的第一个引脚
  3. 选择Edit->Paste Array菜单

    输入引脚间距及需要复制的数量，

    我这里先做一边，将Item Count改为26，间距设为-20。

    然后类推复制其它引脚，

    同样按上一步制作余下的27个引脚。
  4. 在SCH LIBRARY栏中选中器件名称，然后点击Edit
  5. 在弹出的对话框中选择左下角的Edit Pins。
  6. 在name栏中参照DATASHEET依次输入引脚电气名称，注意在引脚电气名称输入过程不要点击鼠标，直接按键盘的↑、↓方向键即可切到下一个引脚的电气名称输入框，修改其它属性也类似。
